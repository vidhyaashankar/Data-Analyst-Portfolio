Introduction

The Uber Trip Analysis dashboard is designed to analyze ride-hailing usage and time analysis patterns for June 2024. Built during data analyst training, the project leverages a dataset of over 103,000 Uber trips to uncover operational insights, demand trends, and geographic patterns. The dashboard offers both high-level summaries and granular drill-down capabilities, making it useful for executives, operations teams, and analysts alike.

Objective

The primary objective of this dashboard is to provide a comprehensive, interactive view of Uber trip data to:

  • Monitor key performance indicators (KPIs): such as total trips, revenue, average fare, trip distance, and trip duration.
  • Identify temporal trends: including peak hours, day-of-week patterns, and weekend vs. weekday differences.
  • Analyze location performance: highlight top pickup/drop-off locations and common origin-destination routes using charts and tables.
  • Enable detailed trip analysis: through drill-through pages for granular, trip-level inspection.

    • Data Cleaning Process

    The raw Uber trip data from Excel files contained inconsistencies, missing values, and mixed data types. The following steps were performed in Power Query Editor to prepare the dataset for analysis:

  • Removed Unnecessary Columns – Deleted irrelevant fields that were not needed for dashboard KPIs (e.g., duplicate trip identifiers, unused location codes).
  • Handled Missing Values – Replaced or removed rows with null values in critical fields such as trip date, fare amount, and location IDs.
  • Corrected Data Types – Converted columns to appropriate data types (e.g., trip date to Date, fare to Decimal, distance to Decimal).
  • Standardized Location Names – Cleaned inconsistent location spellings and ensured uniform naming conventions for pickup/drop-off points.
  • Filtered Outliers – Removed trips with unrealistic fares, zero or negative distances, or excessively long durations that indicated data entry errors.
  • Merged Lookup Tables – Joined the location reference table to enrich trip data with descriptive location details.

    • These steps ensured the dataset was accurate, consistent, and ready for further modeling and visualization in Power BI.

    Data Transformation & Modeling

    Once the data was cleaned, the next step involved preparing it for efficient analysis in Power BI.

  • Fact and Dimension Separation: The trip details table was structured as the primary fact table, containing metrics such as trip distance, trip duration, fare amount, and trip date. Lookup tables, such as the location table, were created as dimension tables to hold reference data for pickup and drop-off locations.
  • Data Modeling:Relationships were established between the fact and dimension tables using unique keys (e.g., location IDs), following a Star Schema design for optimized performance and clarity.
  • Granularity Definition: The trip fact table maintained granularity at the individual trip level, ensuring detailed analysis while enabling roll-ups to monthly, weekly, and daily trends.
  • Calculated Columns & Measures: Key calculated columns (e.g., trip month, trip day name) and DAX measures (e.g., total trips, average fare, total distance) were created to support dynamic filtering, aggregation, and KPI tracking.

    • 1. Overview Analysis

    Purpose: Provide a high-level snapshot of operational KPIs and core business metrics — focused on summary analytics and location performance

    Key visuals and elements in this page:

  • KPI Cards Total Trips, Total Revenue, Avg Fare, Avg Distance, Avg Duration, % Trips with Surge.
  • Demand Distribution: Bar/column charts showing trips by day-of-week and by vehicle type to surface dominant segments.
  • Top Locations : Horizontal bar chart (Top N) and a treemap to show the highest-volume pickup and drop-off locations by count. These visuals replace the need for a geographic map by using clear location names and relative sizing.
  • DOLocation Insights: Matrix or table for most frequent routes (e.g., PULocation -> DOLocation) with trip counts and average durations.
  • Interactivity: slicers for date, vehicle type, payment type, and Top N controls; bookmarks for common executive views (e.g., Weekend Summary, Top Pickup Zones).

    • Key Insight: Midtown and the Upper East/West areas are the highest-volume pickup zones based on Top-N ranking and treemap share. Surge fees materially contribute to revenue (approx. ~15%), identified by comparing total fare and surge aggregates across top locations.

    2. Time Analysis

    Purpose: Explore temporal demand patterns for planning and operations.

  • Line Chart: Daily trips and daily revenue trends across June 2024.
  • Hourly Profile: Column chart for trips by hour of day, highlighting afternoon peaks (13:00–17:00).
  • Weekend vs Weekday Comparison: Visuals or slicers for isolating weekend-heavy patterns and comparing hour-of-day profiles.

    • Key Insight: Saturdays and Sundays show elevated demand; afternoons are consistently busier.

    3. Details (Drill-through Page)

    Purpose: Enable deep-dive into location or time-specific trip data.

  • Drill-through context: Users can drill from Overview or Time Analysis into this page scoped by location or time window.
  • Table Visual: Detailed trip-level data (Trip ID, pickup/drop, duration, distance, fare, surge fee, vehicle, payment type).
  • Supporting charts: Hourly/daily profile charts for the selected context and quick stats (avg fare, median duration).

    • Key Insight: Supports granular analysis for operational troubleshooting (e.g., validate why a specific location shows high average fares or long durations).

    Conclusion

    The Uber Trip Analysis dashboard provided a comprehensive view of trip performance, time-based trends, and location-based insights, enabling data-driven decision-making. Through structured data cleaning, modeling, and DAX-driven KPIs, the project successfully transformed raw trip data into an interactive and user-friendly reporting tool. This project demonstrated key skills in Power BI, including data preparation in Power Query, relational modeling using a star schema, creating meaningful measures, and designing an intuitive dashboard aligned with analytical objectives.
